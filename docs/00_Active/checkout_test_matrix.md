# Матрица критических тестов Checkout

Этот документ содержит список обязательных тестов (регрессионный пакет), которые должны успешно проходить перед каждым Release Candidate (RC). 

## 1. Unit & E2E Tests (Frontend / JS)
*Владелец: Frontend Team*

| Сценарий (Feature) | Тип теста | Описание | Ожидаемый результат |
|--------------------|-----------|----------|----------------------|
| **Mapper/Contract** | JS Unit | Сериализация/десериализация DTO заказа. | Данные маппятся в единый JSON Contract без потерь полей. |
| **Multi-vendor Conflict** | JS Unit | Попытка добавить в корзину товар из другого ресторана. | Срабатывает `multi-vendor detector`, предлагается очистить корзину. |
| **Offline Sync Branch** | JS Unit | Отправка заказа в режиме offline. | Заказ добавляется в локальную очередь (`needs_user_action`), нет дублей при появлении сети. |
| **Empty Cart** | E2E Smoke | Попытка перейти к оформлению с пустой корзиной. | Пользователь перенаправляется на главную/меню с ошибкой. |
| **Unauthorized Recovery** | E2E Smoke | Оформление заказа неавторизованным пользователем. | Вызывается окно авторизации, после успеха заказ продолжается без потери данных. |

## 2. API / Backend Tests (PHPUnit)
*Владелец: Backend Team*

| Сценарий (Feature) | Описание | Ожидаемый результат |
|--------------------|----------|----------------------|
| **Preconditions: Empty Cart** | Вызов API с пустым массивом `items`. | Ошибка 422, `reject_reason = empty_cart`. |
| **Preconditions: Multi-vendor** | Вызов API с товарами от разных вендоров. | Ошибка 422, `reject_reason = multi_vendor`. |
| **Out-of-zone Delivery** | Адрес и координаты клиента вне полигонов заведения. | Ошибка 422, `reject_reason = outside_delivery_zone`. |
| **Idempotency** | Повторный POST запрос с одинаковым `idempotency_key` или `trace_id`. | Заказ не дублируется, возвращается код 200/201 с ID первого созданного заказа. |
| **Invalid Total** | Указанная сумма `total_price` не совпадает с расчетной суммой корзины. | Ошибка 422, `reject_reason = invalid_total`. |
| **Happy Path Order** | Корректный запрос со всеми валидными данными. | Заказ успешно создан (201 Created), в БД сохранена привязка к вендору и пользователю. |
