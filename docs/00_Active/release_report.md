## 4. Rollout Strategy (План Раскатывания)
Рекомендуется использовать стандартную стратегию поэтапного выкатывания (Canary/Phased Rollout):
1. **0-1 Часы:** Слияние RC-ветки в `main`. Запуск деплоя через `php artisan down` (если нужен downtime), `git pull`, установка пакетов, накатывание миграций. Снятие maintenance mode.
2. **1-24 Часы (Soft Launch):** Мониторинг метрик на лояльной аудитории/внутренних сотрудниках. Проверка логов (на предмет всплесков `server_error`, 500 кодов, `invalid_total`).
3. **24-72 Часы (Full Rollout):** Масштабирование на всю клиентскую базу. В случае инцидентов уровня P0 инициируется **Экстренный откат**, описанный в `rc_checklist.md`.
