# Release Definition of Done (DoD)

Этот документ описывает критерии готовности (DoD) для релиза стабилизированного процесса Checkout. Релиз считается успешным и может быть выкачен на production только при выполнении всех нижеперечисленных условий.

## 1. Функциональные критерии (Functional DoD)
- [ ] **Happy Path**: Оформление заказа успешно проходит от корзины до страницы успеха (с созданием записи в БД).
- [ ] **Топ-ошибки обрабатываются корректно**:
  - [ ] Ошибка `MULTI_VENDOR` (товары от разных заведений в корзине).
  - [ ] Ошибка `EMPTY_CART` (попытка оформить пустую корзину).
  - [ ] Ошибка `VALIDATION` (некорректные контакты или данные доставки).
  - [ ] Ошибка `OUT_OF_ZONE` (адрес доставки вне зоны обслуживания).
- [ ] **Оффлайн и очередь**: Оффлайн-очередь синхронизируется при появлении сети без дубликатов заказов (idempotency key работает).

## 2. Нефункциональные критерии (Non-Functional DoD)
- [ ] **Контракт данных**: `OrderContract.md` соблюдается, API и Frontend используют единый формат DTO.
- [ ] **Идемпотентность**: Повторная отправка одного и того же запроса с одинаковым `trace_id` / `idempotency_key` не создает дублей заказа.
- [ ] **Скорость ответа (Performance)**: Эндпоинт создания заказа (`POST /api/v1/orders`) отвечает быстрее 500мс в 95% случаев.
- [ ] **Логирование и PII-safe**:
  - [ ] В логах нет открытых персональных данных (телефонов, точных адресов клиентов, полных имен).
  - [ ] Логируются стандартизированные reason-коды (из `OrderRejectReason` enum).

## 3. Эксплуатационные критерии (Operational DoD)
- [ ] **End-to-End Trace**: `trace_id` корректно прокидывается от UI до API и логируется.
- [ ] **Мониторинг**: Настроены алерты (в логах/Sentry) на всплески:
  - `invalid_total`
  - `outside_delivery_zone`
  - 5xx Server Errors
  - Увеличение числа повторных попыток (retries) у offline sync.
- [ ] **План отката (Rollback Plan)**:
  - [ ] Проверен скрипт отката миграций (`php artisan migrate:rollback`).
  - [ ] Документирован процесс быстрого отката фронтенд-сборки (PWA cache).
- [ ] **Бэкапы**: Свежий бэкап базы данных успешно создан перед деплоем RC.
