🚨 Incident Playbook
INC-1: Checkout Unavailable (P0)
SYMPTOM: Users cannot complete orders
DIAGNOSTICS:
1. Check: php artisan down? → redis/file status
2. Check: DB connections → queue workers alive?
3. Check: Payment gateway health
FIX:
1. php artisan queue:restart
2. Scale workers: php artisan horizon:scale
3. If DB: failover to replica
COMMS:
- Status page: "Checkout maintenance"
- Slack: #incidents
INC-2: Offline Orders Not Syncing (P1)
SYMPTOM: Offline orders stuck in queue
DIAGNOSTICS:
1. Check: Service worker registration
2. Check: Network connectivity
3. Check: sync event in browser console
FIX:
1. Clear SW cache: /offline page → retry
2. Re-register SW manually
3. Force sync: localStorage.setItem('forceSync', '1')
COMMS:
- In-app banner: "Sync your orders"
INC-3: Push Notifications Failing (P1)
SYMPTOM: Users not receiving push
DIAGNOSTICS:
1. Check: VAPID keys in config
2. Check: PushSubscription table
3. Check: WebPush queue logs
FIX:
1. Verify keys: services.push.public_key
2. Resubscribe: POST /api/push/subscribe
3. Retry failed: PushNotificationService::retryFailed()
COMMS:
- Notify affected users via email
INC-4: IDOR/Security Breach (P0)
SYMPTOM: Users seeing other orders
DIAGNOSTICS:
1. Check: Auth::id() in responses
2. Check: Signed URL validation
3. Audit: /api/order/* logs
FIX:
1. HOT FIX: Disable guest tracking FF
2. Revoke all sessions
3. Force re-auth
COMMS:
- Privacy incident report
- Notify affected users within 72h
